🎤 10 questions across 3 difficulty levels. Try to answer before expanding.
Beginner
Q1. What is the difference between if, elif, and else? Beginner ›
One-line answer
They are three parts of the same decision chain. Python checks them in order and runs exactly one branch, then moves on.
  • if starts the chain and checks the first condition. If True, runs its block and skips everything else.
  • elif is "else if" — checked only when the previous condition was False. You can have as many as you need.
  • else is the fallback with no condition. It runs only if every condition above it was False. It is optional.

The key rule: only one branch ever runs. Once Python finds a True condition, it executes that block and ignores all remaining elif/else branches, even if they would also have been True.

⚡ Say it in an interview
if checks first, elif checks next only if previous was False, else is the final fallback. Exactly one branch runs per execution.
Q2. What happens if you forget the colon at the end of an if statement? Beginner ›
One-line answer
You get an immediate SyntaxError. The colon is mandatory — it tells Python a new indented block is about to start.

The colon is not decorative. It is part of Python's syntax for starting any block: if, for, def, class, they all require a colon. Python's parser sees the colon and knows the next indented lines belong to that block.

Without the colon, Python cannot parse the statement and throws: SyntaxError: expected ':'. In recent Python versions it even suggests "did you forget a colon?"

Common trap
Forgetting the colon is one of the most frequent beginner errors. If you ever see a SyntaxError on an if line and the condition looks correct, check for the missing colon first.
Q3. What is a truthy value? Give three examples of falsy values. Beginner ›
One-line answer
Every Python value is either truthy or falsy. Truthy means it behaves like True inside an if. Falsy means it behaves like False.

When you write if x:, Python internally calls bool(x). If that returns True, it enters the block. This is called the value's truth value.

The complete falsy list: False, 0, 0.0, "" (empty string), [] (empty list), {} (empty dict), set() (empty set), None.

Everything else is truthy, including: 1, -1, "hello", "False" (non-empty string!), [0] (list with one item), " " (a space).

Common trap
The string "False" is truthy because it is not empty. The string " " (a space) is truthy for the same reason. Python checks emptiness, not content.
⚡ Say it in an interview
Falsy values are: 0, "", [], {}, None, False. Everything else is truthy. Python checks whether a value is "empty or zero", not what it contains.
Q4. Can you have an if without an else? What happens then? Beginner ›
One-line answer
Yes. else is completely optional. If the condition is False and there is no else, Python simply skips the block and continues.

No error, no output, nothing. Python moves to the next statement after the if block as if it were not there. This is perfectly normal and very common in real code.

Example: if score > 100: print("Invalid") — if score is 80, nothing prints and the program continues normally.

Use else only when you have something specific to do in the False case. Many conditions only care about one outcome.

⚡ Say it in an interview
else is optional. Without it, Python silently skips the if block when the condition is False. No error is raised.
Intermediate
Q5. What is the difference between elif and a second separate if? Intermediate ›
One-line answer
elif only checks if the previous condition was False. A separate if always checks, regardless of what came before.

With elif: all the branches belong to one chain. Once Python finds a True condition, it skips all remaining elif/else branches. Only one branch runs.

With a separate if: it is an entirely independent condition. Python will always evaluate it, even if the first if already ran.

python
score = 95

# Using separate if — prints BOTH "A" and "B"
if score >= 90: print("A")
if score >= 80: print("B")   # also True, also runs

# Using elif — prints only "A"
if score >= 90: print("A")
elif score >= 80: print("B")  # skipped
⚡ Rule of thumb
Use elif when conditions are mutually exclusive (like grade ranges). Use separate if statements when conditions are independent and you want to check all of them.
Q6. What is a ternary expression and when should you avoid it? Intermediate ›
One-line answer
A ternary is a one-line if-else that returns a value: x if condition else y. Use it for simple assignments, avoid it when the logic gets complex.

Unlike a regular if statement, a ternary is an expression — it produces a value. That means you can use it anywhere a value is expected: inside a function call, in an f-string, in an assignment.

python
# Good use: simple, readable
label = "pass" if score >= 50 else "fail"
print(f"Status: {'pass' if score >= 50 else 'fail'}")

# Bad use: nested ternary, hard to read
grade = "A" if s >= 90 else "B" if s >= 80 else "C"  # avoid
When to avoid
Avoid ternary when: there are more than two outcomes, the condition itself is long, or you need to nest ternaries. If you have to re-read it to understand it, use a regular if-else.
Q7. How does short-circuit evaluation work with and and or? Intermediate ›
One-line answer
Python stops evaluating as soon as the result is certain. With and, a False left side means the right side is never checked. With or, a True left side means the right side is never checked.
  • and: if left is False, result is definitely False. Python skips the right side entirely.
  • or: if left is True, result is definitely True. Python skips the right side entirely.

This is not just a performance trick — it is how you write safe conditional checks in Python.

python
# Safe: if lst is empty, lst[0] is never accessed
if lst and lst[0] == "admin":
    print("admin found")

# Default value pattern using or
name = input("Name: ") or "Guest"  # "Guest" if input is empty
⚡ Say it in an interview
Short-circuit evaluation means Python evaluates left to right and stops as soon as the result is known. It is used for safety guards and default value patterns.
Advanced
Q8. What does Python's and actually return? It is not always True or False. Advanced ›
One-line answer
and/or return one of the original operand values, not a boolean. They return whichever value they "stopped at" due to short-circuiting.
  • and returns the first falsy value it finds, or the last value if all are truthy.
  • or returns the first truthy value it finds, or the last value if all are falsy.
python
print(5 and 0)        # 0       (first falsy)
print(5 and 10)       # 10      (last value, all truthy)
print(0 or "hello")  # "hello" (first truthy)
print(0 or "")       # ""      (last value, all falsy)

# Practical use: default fallback
name = input() or "Guest"  # "Guest" if input is empty string
⚡ Say it in an interview
and/or are not boolean operators — they are value-returning operators. They return the value that determined the result, which may not be True or False.
Q9. Why is deeply nested if code a problem, and how do you flatten it? Advanced ›
One-line answer
Deep nesting is hard to read and test. The fix is to use guard clauses: check for the invalid case early and return, keeping the main logic flat.

Every extra level of nesting pushes the real logic further right and forces the reader to mentally track multiple open conditions at once. This is sometimes called "arrow code" because of the shape it creates.

python
# Deep nesting — hard to follow
def process(user):
    if user.logged_in:
        if user.is_admin:
            if user.has_permission:
                do_the_thing()

# Guard clauses — flat and readable
def process(user):
    if not user.logged_in: return
    if not user.is_admin:  return
    if not user.has_permission: return
    do_the_thing()   # reached only when all checks pass
⚡ Say it in an interview
Guard clauses invert the condition and return early. This eliminates nesting and puts the happy path at the outermost indentation level, making code easier to read and test.
Q10. How would you replace a long if-elif chain with a dictionary? Advanced ›
One-line answer
When each branch maps one value to another, replace the chain with a dict and use .get(key, default). Shorter, faster, and easier to extend.

A long if-elif chain that maps one value to another is really just a lookup table in disguise. A dictionary expresses that mapping directly.

python
# if-elif chain — verbose, hard to extend
if   day == "Mon": label = "Monday"
elif day == "Tue": label = "Tuesday"
elif day == "Wed": label = "Wednesday"
# ... 4 more lines

# Dictionary version — clean, easy to extend
days = {"Mon": "Monday", "Tue": "Tuesday", "Wed": "Wednesday"}
label = days.get(day, "Unknown")  # "Unknown" if key not found
When this does NOT work
Dict lookup only works when each branch returns a value. If each branch runs different code (not just a value assignment), use a dict of functions or keep the if-elif.
⚡ Say it in an interview
When an if-elif chain is just mapping one value to another, a dictionary with .get() is cleaner, O(1) lookup instead of O(n), and easier to extend without touching logic.
⚠️ These are the mistakes almost every beginner makes with conditionals. Read through them once and you'll save yourself hours of debugging.
1
Forgetting the colon at the end of if / elif / else
SyntaxError
›
Why it happens: Coming from math or other languages, the colon feels redundant. But Python uses the colon to say "a new block starts after this line." Without it, Python cannot even parse what you wrote. It crashes before running a single line.
❌ Wrong
python
score = 95
if score >= 90 
    print("A grade")
elif score >= 80 
    print("B grade")
else 
    print("Try again")
Python says
  File "main.py", line 2
    if score >= 90
                  ^
SyntaxError: expected ':'
✅ Correct
python
score = 95
if score >= 90:
    print("A grade")
elif score >= 80:
    print("B grade")
else:
    print("Try again")
🧠 Memory Trick
Read the if line out loud as a sentence: "If score is 90 or more..." The colon is the pause after that sentence. It means "then do this." If you can hear the pause, you will remember the colon.
⚡ Key Rule
Every if, elif, and else line must end with a colon :. No exceptions. Python even tells you exactly where it is missing: look at the arrow ^ in the error message.
2
Wrong indentation makes code run outside the if block
Logic Error
›
Why it happens: This one is sneaky because Python does not crash. It silently runs the misindented line every single time, regardless of the condition. You only notice when you see unexpected output.
❌ Wrong:"Congrats!" always runs
python
score = 40
if score >= 50:
    print("You passed!")
print("Congrats!")   # NOT indented
Actual output (score = 40)
Congrats!
(You passed! never printed, but Congrats! did!)
✅ Correct:both lines inside
python
score = 40
if score >= 50:
    print("You passed!")
    print("Congrats!")   # indented = inside
Actual output (score = 40)
(nothing, score is less than 50 so both lines are skipped)
🧠 Memory Trick
Think of the if block as a room. Every line you want inside the room must step 4 spaces to the right. If a line steps back to the wall (column zero), it is standing outside the room and runs no matter what.
⚡ Key Rule
In Python, indentation is the block. There are no curly braces. Python uses spacing alone to group lines. Every line inside an if must be indented by exactly 4 spaces. One space off and it is a completely different program.
3
Breaking the if-elif-else chain with a stray statement
SyntaxError
›
Why it happens: You add a quick debug print between the if and elif to see what is happening. Looks harmless. But Python sees it as the end of the if block. So when it hits elif next, it thinks: "elif without an if? That makes no sense."
❌ Wrong:stray print breaks the chain
python
if x > 0:
    print("positive")

print("checking...")  # this ends the chain

elif x == 0:
    print("zero")
Python says
  File "main.py", line 6
    elif x == 0:
    ^^^^
SyntaxError: invalid syntax
✅ Correct:nothing between them
python
if x > 0:
    print("positive")
elif x == 0:
    print("zero")
else:
    print("negative")
🧠 Memory Trick
Think of if, elif, else as train coaches. They must be coupled directly together. A stray print between them is like cutting the coupling wire. The train falls apart. Keep your debug prints before the if or after the entire else.
⚡ Key Rule
if, elif, and else must be written back-to-back with nothing in between. They are one connected structure. Any statement between them tells Python the chain is over, and the next elif or else becomes orphaned.
4
Using = (store) instead of == (compare) inside a condition
SyntaxError
›
Why it happens: In everyday English, "if grade = A" sounds perfectly natural. But Python treats = as a storage action: "put A into grade." That makes no sense inside a condition. Python expects a question, not an assignment.
❌ Wrong:single = is not a question
python
grade = "A"
if grade = "A":
    print("Excellent!")
Python says
  File "main.py", line 2
    if grade = "A":
             ^
SyntaxError: invalid syntax. Maybe you meant '==' ?
✅ Correct:== asks a question
python
grade = "A"
if grade == "A":
    print("Excellent!")
# Output: Excellent!
=
ASSIGNMENT
"Store this value"
Used outside conditions
==
COMPARISON
"Are these equal?"
Used inside conditions
🧠 Memory Trick
One = is a mouth open to receive something (you're storing). Two == are two eyes looking at each other to see if they match (you're comparing). Inside an if, you're always looking, never storing.
⚡ Key Rule
Inside any condition (if, elif, while), always use == to compare. Python 3 is kind enough to suggest == in the error message when it catches this. Future Python versions may not be as forgiving.
5
Thinking the string "0" or "False" is falsy
Logic Error
›
Why it happens: You learn that 0 is falsy and assume "0" (the string) is too. But Python does not read the content of a string to judge it. It only asks: is this string empty? "0" has one character, so no, it is not empty — it is truthy.
❌ Wrong assumption
python
answer = "0"   # came from input()
if not answer:
    print("No answer")
else:
    print("Got answer")
Actual output (unexpected!)
Got answer
("0" is truthy, it is NOT empty!)
✅ Correct:convert, then check
python
answer = "0"   # came from input()
if int(answer) == 0:
    print("Answer is zero")
else:
    print("Non-zero")
Actual output (correct!)
Answer is zero
What is actually falsy in Python?
0 0.0 "" [] {} set() None False
These look truthy but are NOT: "0", "False", "None". They are non-empty strings, so Python treats them as truthy.
🧠 Memory Trick
Python is like a bouncer checking if a bag is empty, not reading what is written on it. A bag with the word "empty" written on it is still not an empty bag. "0" is a bag with something in it (the character 0), so Python calls it truthy.
⚡ Key Rule
Python judges values, not text. The number 0 is falsy. The string "0" is truthy. If a value came from input(), it is always a string. Convert it with int() or float() before doing any numeric truthiness check.
Key Takeaways
🔀ifstatement
  • Runs only when condition is True
  • Skips the block entirely when False
if score >= 50: ...
↔️if-elsestatement
  • Always picks exactly one path
  • One block runs, the other is skipped
if x > 0: ... else: ...
🔗if-elif-elsestatement
  • Checks conditions top to bottom
  • Stops at the first True one
  • Only one block ever runs
if a: ... elif b: ... else: ...
📦Nested ifstatement
  • An if placed inside another if
  • Inner check only runs when outer is True
  • Both must be True to reach inner block
if logged_in: if is_admin: ...
⚡Ternaryexpression
  • One-liner that picks between two values
  • Great for simple assignments
  • Avoid for complex logic, gets hard to read
label = "pass" if score >= 50 else "fail"
🔍Truthy / Falsyconcept
  • Any value can go inside an if
  • Falsy: 0, "", [], {}, None, False
  • "0" and "False" are truthy (non-empty strings)
if name: # True only if name is not empty
📐Indentationrule
  • Spacing defines blocks, not braces
  • Every line inside must be 4 spaces in
  • One wrong indent silently breaks logic
if True: print("inside")
🏁First match winsrule
  • Python stops at the first True branch
  • Remaining branches are never checked
  • Even if later branches would also be True
if score >= 90: ... # checked first

Cheat Sheet
python
if condition:
    # runs when True
    # skipped entirely when False
Use when you only care about the True case and want to do nothing otherwise.
python
if condition:
    # runs when True
else:
    # runs when False
Always picks one path. One of the two blocks always runs, never both.
python
if cond1:
    # runs if cond1 is True
elif cond2:
    # runs if cond1 is False AND cond2 is True
elif cond3:
    # runs if both above are False AND cond3 is True
else:
    # runs if ALL conditions above are False
Python stops at the first True condition. The remaining branches are never checked.
python
if outer_condition:
    if inner_condition:
        # runs only when BOTH are True
The inner if is only checked if the outer one is already True.
python
result = "yes" if condition else "no"

# real example
label = "pass" if score >= 50 else "fail"
One-liner shorthand for simple if-else. Good for assignments, not for complex logic.
python
# Falsy: treated as False inside any if
0  0.0  ""  []  {}  set()  None  False

# Truthy: everything else
1  -5  "hello"  "0"  [1,2]  True
Strings with content are always truthy, even "0" or "False". Python checks emptiness, not what is written inside.
