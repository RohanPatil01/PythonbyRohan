How Python Runs

🔄 Python Reads Your Code Top to Bottom

Before anything else, let's understand one golden rule: Python reads and runs your code line by line, from top to bottom. It does not jump around. It does not read ahead. Whatever comes first runs first.

python 
print("Line 1 runs first")
print("Line 2 runs second")
print("Line 3 runs last")
💡 This means if you use a variable before you create it, Python will throw an error. Order matters. Always.
Definition

🐍 What is Python?

Python is a high-level, interpreted, general-purpose programming language created by Guido van Rossum and first released in 1991. The name "Python" was inspired by the British comedy group Monty Python, not the snake.

Today Python is one of the most popular languages in the world. It is used in web development, data science, artificial intelligence, automation, cybersecurity, and much more. Companies like Google, Netflix, Instagram, and ISRO use Python.

Think of it this way

💬 Why Python Feels Easy

Think of programming languages like spoken languages. C++ is like German, powerful and precise but takes years to get fluent. Java is like French, beautiful structure but a lot of grammar rules. Python is like English. It is not perfect, but almost everyone can read it and make sense of it on the first try.

Think about the first time you read a Python program versus a C++ program. Python reads almost like plain English instructions. That is exactly what makes it the best first language for most learners.

Why this matters

🤔 Why Should You Learn Python?

  • Simple syntax:reads like plain English, great for beginners
  • Versatile:data science, web, AI, automation, all in one language
  • Huge demand:top skill at TCS, Infosys, Wipro, and every data-focused startup
  • Great community:millions of resources, Stack Overflow answers, YouTube tutorials
  • Free and open source:zero cost, runs on any operating system
Syntax

🖨️ The print() Function

Let's start with the most satisfying thing in programming: making the computer say something. The print() function displays output on your screen. Every single Python programmer learns this first.

python 
print("Hello, World!")
print("Hello", "World")            # prints: Hello World
print("Hello", "World", sep="-")  # prints: Hello-World
print("Hello", end=" ")           # no newline at end
print("World")                     # prints on same line: Hello World
💡 By default print() adds a new line at the end. Use end="" to stop that. Use sep= to control what goes between multiple values.
Syntax

🔀 Escape Characters

Sometimes you need to include special characters inside a string that Python would normally misread. That is where escape characters come in. They always start with a backslash \.

python 
print("Line 1\nLine 2")        # \n:new line
print("Name:\tRohan")           # \t:tab space
print("He said \"Hello\"")       # \":double quote inside string
print("C:\\Users\\Student")        # \\:literal backslash
print('It\'s easy')            # \':single quote inside string
📝 Real-world use: \t is great for aligning bill receipts or tables in the terminal. \n is how you print multi-line messages in one print statement.
Concept

💬 Comments

Comments are lines in your code that Python completely ignores when running. They are notes you write for yourself and other humans.

Here is a real scenario: a student writes a complex data analysis script for a college project. Three months later, when a classmate asks to use it, even the original author cannot remember what half the code does. That is exactly why comments exist. They are notes to your future self.

python 
# This is a single-line comment

print("Hello")   # This is an inline comment

"""
This is a multi-line comment.
Also called a docstring.
Python does not run any of this.
"""
💡 Good rule: write a comment when the code is not obvious. You do not need to comment x = 5 with "assigning 5 to x". But do comment a 10-line calculation with what it is computing.
Definition

📦 Variables

A variable is a named container that stores a value in memory. You give it a name, assign it a value, and Python remembers it for you throughout the program.

In Python you do not need to declare the type of variable. Python is smart enough to figure it out based on the value you give it. This is called dynamic typing.

python 
name   = "Rohan Patil"  # string
age    = 24             # integer
cgpa   = 10.0           # float
is_pass    = True           # boolean

# Multiple assignment in one line
x, y, z = 1, 2, 3

# Same value to multiple variables
a = b = c = 0

print(name, age)   # Rohan Patil 24
Think of it this way

🏷️ Variables are Like Labelled Boxes

Imagine you are packing for a trip. You have a box, and you stick a label on it that says "clothes". Now whenever you need clothes, you look for the box labelled "clothes". You do not need to remember exactly where the clothes are physically stored in the room. The label handles that.

Variables work the same way. When you write name = "Rohan", you are creating a box in memory, putting "Rohan" inside it, and sticking a label called name on it. Whenever your program needs that value, it looks for the name label.

Rules and Conventions

📏 Variable Naming Rules and snake_case

The hard rules (break these and Python throws an error):

  • Must start with a letter or underscore, not a number
  • Can contain letters, numbers, and underscores only
  • No spaces allowed
  • Case-sensitive: name and Name are two different variables

The convention (break these and your code still works, but other programmers will judge you):

Python programmers follow snake_case for variable names. This means all lowercase letters with underscores between words. Like student_name, total_marks, is_eligible. Why? Because Python's own standard library uses snake_case everywhere. Following this makes your code look consistent with professional Python code.

python 
# Valid and professional (snake_case)
student_name  = "Sagar"
total_marks   = 487
is_eligible   = True

# Valid but not Pythonic
StudentName   = "Sagar"   # PascalCase (used for classes)
totalMarks    = 487      # camelCase (used in JavaScript)

# Invalid (will throw SyntaxError)
# 1name = "bad"
# my-name = "bad"
# my name = "bad"
⚠️ Reserved Keywords. These words are reserved by Python and cannot be used as variable names: if, else, for, while, def, class, return, True, False, None, import, and, or, not. Python uses these words for its own grammar.
Why this rule exists

🤔 Why Can't Variable Names Start With a Number?

Great question. If you wrote 1name = "Rohan", Python would read the 1 and think you are writing a number. Then it sees name right after and gets confused because a number cannot be followed by letters without an operator between them.

Basically Python's brain (the parser) reads numbers and names using different rules. Mixing them at the start creates an ambiguity Python cannot resolve. So the rule is simple: start with a letter or underscore, never a digit.

Syntax

⌨️ The input() Function

So far we have been printing things. Now let's make Python listen. The input() function pauses your program, waits for the user to type something and press Enter, then gives you whatever they typed.

python 
name = input("Enter your name: ")
print("Hello,", name)

# Always convert to number if you need math
age  = int(input("Enter your age: "))
cgpa       = float(input("Enter your CGPA: "))
Think of it this way

📞 Why input() Always Returns a String

Here is the trick that confuses almost every beginner. input() always gives you a string, no matter what the user types. Even if they type 25, Python gives you "25" (with invisible quote marks around it), not the number 25.

Think of it like a phone call. When someone speaks to you over the phone, everything arrives as sound waves. Whether they said a number, a name, or a poem, it all arrives the same way. You have to interpret it yourself. Similarly, input() brings everything in as text. You have to convert it to a number if that is what you need.

That is why we write int(input(...)). The int() is you "interpreting" the text as a number.

Syntax

🔁 Type Conversion

Python lets you convert one data type into another. This is called type conversion or type casting. You will use this constantly, especially right after input().

python 
# str to int
age = int("24")         # 24

# str to float
cgpa       = float("10.0")   # 10.0

# int to str
code = str(2001)        # "2001"

# float to int (drops decimal, does NOT round)
marks = int(95.9)       # 95, not 96

print(type(age))        # <class 'int'>
⚠️ ValueError alert. If the user types "twenty" when you do int(input(...)), Python throws a ValueError because "twenty" cannot be converted to a number. You will learn how to handle this gracefully in the Exception Handling topic.
Syntax

🔤 f-Strings (Formatted Strings)

f-strings are the modern, clean way to embed variables directly inside a string. Prefix the string with f and put variable names inside curly braces. Python replaces {variable} with the actual value when it runs.

python 
name    = "Rohan Patil"
college = "B. N. Bandodkar College of Science"
cgpa    = 10.0

print(f"Name   : {name}")
print(f"College: {college}")
print(f"CGPA   : {cgpa}")

# Expression inside f-string
print(f"10 + 5 = {10 + 5}")

# Formatting decimal places
print(f"Pi is approximately {22/7:.4f}")
💡 f-strings were introduced in Python 3.6. They are faster and more readable than older methods like % formatting or .format(). Always use f-strings for new code.
Syntax

🔍 type() and id() Functions

type() tells you what kind of data a variable holds. id() tells you the memory address where Python stored the value.

python 
age  = 24
name = "Rohan"
cgpa = 10.0

print(type(age))   # <class 'int'>
print(type(name))  # <class 'str'>
print(type(cgpa))  # <class 'float'>

print(id(age))     # something like 140234567
print(id(name))    # a different memory address
💡 When is id() actually useful? Imagine two variables seem equal. id() tells you if they are pointing to the exact same object in memory or two different objects with the same value. This becomes important when you start working with lists and mutable objects.

Beginner Level
Q1. What is Python? Name some of its key features. Beginner ›
One-line answer
Python is a high-level, interpreted, general-purpose programming language known for its simple, readable syntax and massive ecosystem.

Created by Guido van Rossum and released in 1991, Python is today one of the most popular languages in the world. The key features interviewers love to hear:

  • Easy syntax: reads almost like plain English, minimal boilerplate
  • Dynamically typed: no need to declare variable types explicitly
  • Interpreted: runs line by line, errors appear immediately
  • Object-oriented: supports classes, inheritance, polymorphism
  • Large standard library: batteries included, hundreds of built-in modules
  • Cross-platform: same code runs on Windows, Mac, and Linux
⚡ Say it in an interview
Python is high-level, interpreted, and dynamically typed. Its biggest strengths are readable syntax, a huge standard library, and versatility across web, data science, AI, and automation.
Q2. What is the difference between print() and input() in Python? Beginner ›
One-line answer
print() sends data out to the screen. input() brings data in from the user and always gives it back as a string.
  • print(value): displays output. You can pass multiple values, control separators with sep=, and control the line ending with end=.
  • input("prompt"): pauses the program, shows the prompt, waits for the user to type and press Enter, then returns what they typed as a string.

The most important thing about input(): it always returns a string, no matter what the user types. Even if they type 42, Python gives you "42". You must wrap it with int() or float() to use it as a number.

Common trap
Forgetting to convert input() to a number before doing math. age + 1 will throw a TypeError if age is still a string. Always use int(input(...)) when you need a number.
Q3. Is Python case-sensitive? Beginner ›
One-line answer
Yes, completely. name, Name, and NAME are three separate variables. Python treats every character as distinct.

Case sensitivity applies everywhere in Python: variable names, function names, class names, and keywords. This means:

  • True is valid. true raises a NameError.
  • None is valid. none raises a NameError.
  • print("hi") works. Print("hi") raises a NameError.

This also means snake_case matters: student_name and Student_Name are completely different variables even though they look related.

⚡ Say it in an interview
Yes, Python is fully case-sensitive. Keywords like True, False, and None must be capitalized exactly as shown. Variable names follow the same rule.
Q4. What are variables in Python? How are they different from other languages? Beginner ›
One-line answer
A variable is a named label that points to a value in memory. In Python, you never declare the type. Python figures it out from the value you assign.

In C or Java, you must declare the type before using a variable: int age = 24;. In Python, you just write age = 24 and Python infers the type automatically. This is called dynamic typing.

Another Python-specific behaviour: you can reassign a variable to a completely different type at any time:

  • x = 10 makes x an integer
  • x = "hello" now makes x a string. No error, no redeclaration needed.

Under the hood, Python variables are just names that point to objects. When you reassign, the name now points to a different object.

⚡ Say it in an interview
Python is dynamically typed. Variables are labels pointing to objects, not typed containers. The type is attached to the object, not the variable name.
Intermediate Level
Q5. What is the difference between = and == in Python? Intermediate ›
One-line answer
= assigns a value to a variable. == compares two values and returns True or False. They do completely different things.
  • x = 5: stores the value 5 in the variable x. This is a statement, not an expression. It does not return anything.
  • x == 5: checks whether x currently holds the value 5. Returns True or False. Used inside conditions.

Example: if x = 5: causes a SyntaxError. You cannot use = inside a condition. Python requires == for comparison.

Common trap
Using = inside an if condition by accident: if x = 10:. Python raises a SyntaxError here. In languages like C this is a silent bug, but Python prevents it entirely.
Q6. What are f-strings and when were they introduced? Intermediate ›
One-line answer
f-strings let you embed variables and expressions directly inside a string using {}. They were introduced in Python 3.6 and are now the preferred way to format strings.

Before f-strings, developers used % formatting ("Hello %s" % name) or .format() ("Hello {}".format(name)). Both are verbose and easy to mess up with multiple values.

f-strings are faster, shorter, and easier to read. You prefix the string with f and put any Python expression inside {}:

  • f"Hello {name}": embed a variable
  • f"Result: {10 + 5}": embed an expression
  • f"Pi: {22/7:.4f}": format decimal places
⚡ Say it in an interview
f-strings (formatted string literals) were introduced in Python 3.6. They are the fastest and most readable string formatting method. Always prefer them over % or .format().
Q7. What is the difference between compiled and interpreted languages? Intermediate ›
One-line answer
Compiled languages translate all code to machine language before running. Interpreted languages like Python execute code line by line at runtime.
  • Compiled (C, C++, Go): the entire source code is converted to machine code by a compiler before execution. The resulting binary is fast to run but takes time to compile.
  • Interpreted (Python, Ruby): an interpreter reads and executes the code line by line at runtime. No compilation step needed, but generally slower for heavy computation.

Python actually does compile to bytecode (.pyc files) internally, then the CPython interpreter runs that bytecode. It is both compiled and interpreted in a sense, but the user-facing experience is that of an interpreted language.

⚡ Say it in an interview
Python is interpreted: code runs line by line. This makes debugging easier (errors appear immediately) but means Python is slower than compiled languages for CPU-heavy tasks.
Advanced Level
Q8. What is dynamic typing? What are the trade-offs? Advanced ›
One-line answer
Dynamic typing means the type of a variable is determined at runtime, not declared in advance. Faster to write, but type bugs may only appear when the code actually runs.

Advantages:

  • Faster to write: x = 5 instead of int x = 5;
  • More flexible: a variable can hold any type at any point
  • Less boilerplate, easier to prototype

Disadvantages:

  • Type errors surface at runtime, not at compile time. A bug in rarely-run code can go unnoticed for a long time.
  • IDEs and linters have less certainty about types, making autocomplete less reliable.

Python 3.5 introduced optional type hints (def greet(name: str) -> str:) to get the benefits of type annotations without giving up dynamic typing.

⚡ Say it in an interview
Dynamic typing trades compile-time safety for development speed. Type hints (Python 3.5+) are the best of both worlds: optional annotations that tools can check without the language enforcing them.
Q9. What is the difference between mutable and immutable objects? Advanced ›
One-line answer
Mutable objects can be changed after creation. Immutable objects cannot. Modifying an immutable object always creates a new one instead.
  • Mutable: list, dict, set. You can add, remove, or change elements in place.
  • Immutable: int, float, str, tuple. Any "modification" creates a brand new object. The original is untouched.

Example: s = "hello"; s += " world" does not modify the original string. Python creates a new string "hello world" and makes s point to it. The old "hello" object still exists in memory until the garbage collector cleans it up.

This matters most when passing objects to functions: mutable objects passed to a function can be modified inside it, affecting the caller. Immutable objects cannot.

Common trap
Using a mutable object (like a list) as a default function argument. Python creates the default once at definition time, so all calls share the same list object. Use None as default and create the list inside the function instead.
Q10. How does Python manage memory? Advanced ›
One-line answer
Python uses automatic memory management via reference counting. When no variable points to an object, the memory is freed automatically.

Python's memory management has two main mechanisms:

  • Reference counting: every object in memory has a counter tracking how many variables point to it. When that count drops to zero (no references left), Python immediately frees the memory.
  • Cyclic garbage collector: reference counting alone cannot handle circular references (object A points to B, B points back to A). Python's cyclic GC periodically scans for and cleans up these cycles.

You can inspect memory details using: id(x) shows the memory address of x, and sys.getrefcount(x) shows how many references point to x.

⚡ Say it in an interview
Python uses reference counting for immediate deallocation and a cyclic garbage collector for circular references. Developers never call free() manually. Memory is managed entirely by the runtime.
💡 Every programmer makes these mistakes at the start. Knowing them in advance puts you ahead of 90% of beginners.
1
print without parentheses
SyntaxError
›
Why it happens: In Python 2, print was a statement and worked without parentheses. In Python 3, it became a function. If you forget the parentheses, Python 3 has no idea what you mean and crashes immediately.
❌ Wrong
python
print "Hello"
Python says
  File "main.py", line 1
    print "Hello"
          ^^^^^^^
SyntaxError: Missing parentheses in call to 'print'
✅ Correct
python
print("Hello")
🧠 Memory Trick
In Python 3, print is just a regular function, exactly like input() or len(). You always call functions with parentheses. No parentheses means you are just looking at the function, not calling it.
⚡ Key Rule
Python 3 requires print() with parentheses, always. Python even tells you exactly this in the error message: "Missing parentheses in call to 'print'."
2
Forgetting to convert input() to a number
Logic Error
›
Why it happens: input() always returns a string, no matter what the user types. Even if they type 5, you get the text "5", not the number 5. So adding two inputs concatenates them as text instead of doing math.
❌ Wrong
python
a = input("Enter a: ")
b = input("Enter b: ")
print(a + b)
User types 5 and 3, but gets:
53
(strings joined, not numbers added!)
✅ Correct
python
a = int(input("Enter a: "))
b = int(input("Enter b: "))
print(a + b)
User types 5 and 3, now gets:
8  (correct!)
🧠 Memory Trick
Think of input() as a postman who only delivers letters (strings). Even if you write "42" on the letter, it is still a letter, not a number. You have to open the letter and convert the content yourself using int() or float().
⚡ Key Rule
input() always returns a string. Wrap it with int() for whole numbers or float() for decimals whenever you need to do math with user input.
3
Variable name starting with a number
SyntaxError
›
Why it happens: Python uses the first character of a name to decide what it is looking at. If the first character is a digit, Python thinks it is seeing a number literal, not a variable name. The rest of the line then makes no grammatical sense, so it crashes.
❌ Wrong
python
1name = "Rohan"
2score = 95
Python says
  File "main.py", line 1
    1name = "Rohan"
     ^^^^
SyntaxError: invalid syntax
✅ Correct
python
name1     = "Rohan"
score_2   = 95
_private  = "ok too"
🧠 Memory Trick
Think of a variable name like a person's name. Names start with a letter, not a number. You would never call someone "1John" but you might call them "John1". Same rule applies in Python.
⚡ Key Rule
Variable names must start with a letter (a-z, A-Z) or an underscore (_). Numbers are allowed anywhere after that. No spaces, no hyphens, no starting digit.
4
Using = (store) instead of == (compare) in a condition
SyntaxError
›
Why it happens: In everyday English, "if x = 5" sounds natural. But in Python, = is a storage action (put 5 into x). Inside a condition, Python expects a question (is x equal to 5?). That question is written with ==.
❌ Wrong
python
if x = 5:
    print("five")
Python says
  File "main.py", line 1
    if x = 5:
         ^
SyntaxError: invalid syntax. Maybe you meant '==' ?
✅ Correct
python
if x == 5:
    print("five")
🧠 Memory Trick
One = is a mouth open to receive something (storing). Two == are two eyes looking at each other to check if they match (comparing). Inside an if, you are always comparing, never storing.
⚡ Key Rule
= assigns a value. == compares two values. Inside if, elif, and while, always use ==.
5
Using a variable before creating it
NameError
›
Why it happens: Python runs your code top to bottom, one line at a time. It has no ability to look ahead. If you try to use a variable before the line that creates it, Python genuinely has never seen that name before and raises a NameError.
❌ Wrong
python
print(score)   # used before created
score = 95
Python says
NameError: name 'score' is not defined
✅ Correct
python
score = 95      # create first
print(score)   # then use: prints 95
🧠 Memory Trick
Python is like a chef who only cooks with ingredients already on the counter. If you ask for a dish before putting the ingredients out, the chef cannot start. You must put the ingredients out (create the variable) before asking for the dish (using it).
⚡ Key Rule
Always assign before you use. The line that creates a variable must come before any line that reads it. Python runs top to bottom with no lookahead.
Key Takeaways
🖨️print()built-in
  • Displays output to the screen
  • Use sep= to change what goes between items
  • Use end= to change what prints at the end (default is newline)
print("a", "b", sep="-", end="!")
⌨️input()built-in
  • Reads what the user types
  • Always returns a string, no exceptions
  • Wrap with int() or float() to get a number
age = int(input("Age: "))
📦Variablesconcept
  • Named labels that point to values
  • Dynamically typed: no need to declare the type
  • Use snake_case by convention
user_name = "Rohan"
🔤f-stringssyntax
  • Prefix string with f, put variables in {}
  • Can embed any expression, not just variables
  • Use {value:.2f} to format decimal places
print(f"Hi {name}, score: {score:.1f}")
🔀Escape characterssyntax
  • \n inserts a new line
  • \t inserts a tab
  • \" puts a quote inside a string
print("Line1\nLine2\tindented")
🔁Type conversionconcept
  • int(), float(), str() convert between types
  • Raises ValueError if the string is not a valid number
  • int("3.14") fails; use int(float("3.14")) instead
x = int(float("3.14")) # 3
🔄Execution orderrule
  • Python reads top to bottom, one line at a time
  • A variable must be created before it can be used
  • Order of your code always matters
x = 5 # must come before print(x)
💬Commentssyntax
  • # starts a single-line comment
  • Triple quotes """...""" for multi-line notes
  • Ignored by Python, only for humans reading the code
# this is a note for whoever reads the code
