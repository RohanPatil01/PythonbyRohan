Operators in Python
Operators are the symbols that tell Python what to do with your data. From simple addition to comparing values and combining conditions, operators are everywhere in your code.

💡 You know variables store data. Now let's actually do stuff with that data. Operators are just symbols that tell Python what to do: add these two numbers, check if this value is bigger, see if this item is in that list. You use them in literally every program you write.
Python Operators: Overview

➕ Arithmetic Operators

Most of these you already know from school. The only two that are new are // and %. They confuse almost every beginner at first, so let's go slow on those.

OperatorNameInput typesOutput typeExampleResult
+Additionint / floatint or float10 + 313
-Subtractionint / floatint or float10 - 37
*Multiplicationint / floatint or float10 * 330
/Divisionint / floatalways float10 / 25.0
//Floor Divisionint / floatint (if both int)10 // 33
%Modulusint / floatint or float10 % 31
**Exponentint / floatint or float2 ** 8256
🔍 What does // actually do? Imagine you have 10 chocolates and 3 friends. You give each friend as many as possible equally. Each friend gets 3 chocolates (3 x 3 = 9). The 1 leftover chocolate is not shared. So 10 // 3 = 3. The answer is always how many each person gets, never the leftover.
💡 What does % actually do? Now flip it. Instead of asking "how many each?" you ask "how much is left over?" With 10 chocolates and 3 friends, 1 chocolate is left. So 10 % 3 = 1. That leftover is all % cares about. Real use: n % 2 tells you if a number is even (0 leftover) or odd (1 leftover).
⚠️ Watch out: / always gives a decimal answer. Even 10 / 2 gives 5.0, not 5. If you want a whole number with no decimal point, use //.

⚖️ Comparison Operators

These always answer a yes/no question. Is this bigger? Are these equal? The answer is always True or False. That's exactly what an if statement or while loop needs to decide what to do next.

💡 The output is always True or False. So you can drop any comparison directly into an if or while: if age >= 18: or while score < 100:. Python checks it and decides yes (True) or no (False).
OperatorMeaningInput typesOutput typeExampleResult

!=Not equal toanybool5 != 3True

<Less thanint / float / strbool3 < 5True
>=Greater than or equalint / float / strbool5 >= 5True
<=Less than or equalint / float / strbool4 <= 5True
⚠️ = and == are completely different things. = puts a value into a variable: x = 5. == asks "are these two things equal?": x == 5. If you write if x = 5: Python throws an error immediately. Inside any condition, always use ==.

Bonus Python trick: You can write 0 <= score <= 100 directly, just like you'd write it in maths. In most other languages you'd have to write score >= 0 and score <= 100. Python's version is shorter and easier to read.

🧠 Logical Operators

Sometimes one condition is not enough. What if you need "age must be over 18 AND they must have a valid ID"? That's where logical operators come in. They let you connect multiple conditions into one.

💡 These also give True or False as output. So you use them the same way as comparison operators: inside if and while. You'll often chain them: if age >= 18 and has_id: means both things must be true.
OperatorMeaningInput typesOutput typeExampleResult
andBoth must be TrueboolboolTrue and FalseFalse
orAt least one TrueboolboolTrue or FalseTrue
notFlips True to Falseboolboolnot TrueFalse

Python stops checking early when it already knows the answer

Here's something clever Python does. When you use and or or, Python doesn't always check both sides. If it can figure out the final answer from just the left side, it stops right there and skips the right side completely.

and With and, BOTH sides must be True for the result to be True. So if the left side is already False, the answer is definitely False no matter what the right side says. Python skips the right side. Example: False and some_function() — Python never even calls some_function().
or With or, just ONE side needs to be True. So if the left side is already True, the answer is definitely True. Python skips the right side. Example: True or some_function() — Python never calls some_function().
not not just flips the answer. not True becomes False. not False becomes True. It always checks what's after it, no skipping here.

Why should you care about this?

python
# Imagine user is None (no user logged in).
# If we check user.name directly, Python crashes because None has no .name.
# But the left side (user is not None) is False, so Python skips the right side.
# No crash. This is a very common safety pattern.
if user is not None and user.name == "Rohan":
    print("Hello Rohan!")

# If input() gives back an empty string (user just pressed Enter),
# the or kicks in and name becomes "Guest" instead.
name = input("Name: ") or "Guest"

📌 Assignment Operators

You already know =. The rest are just shortcuts. score = score + 10 and score += 10 do the exact same thing. The shortcut version just saves you from typing the variable name twice.

💡 One thing to watch: x /= 2 will always make x a decimal (float), even if the result is a whole number. That's because / always produces a float. All the other shortcuts keep the same type as the original value.

🔍 Identity Operators

Here's the difference: == asks "do these two things have the same value?" while is asks "are these literally the same object?" Imagine printing two copies of the same document. The text on both copies is identical (== is True), but they are physically two different sheets of paper (is is False).

💡 When do you actually use this? Almost only for checking None. When a function finds nothing to return, it returns None. You check it with if result is None:. That's the main real-world use.
OperatorMeaningInput typesOutput typeExample
isSame object in memory?anyboolx is None

python
a = [1, 2, 3]
b = [1, 2, 3]
print(a == b)   # True: same values
print(a is b)   # False: two separate lists in memory

c = a            # c and a now point to the SAME list
print(a is c)   # True: same object
⚠️ Stick to is only when checking None, True, or False. For everything else, use ==. Using is to compare numbers or strings can give you unexpected results depending on how Python stores them internally.

🎯 Membership Operators

These answer one simple question: "is this thing inside that collection?" You give it a value and a list (or string, or set), and it tells you yes or no. Very useful for things like checking if a username already exists, or if a word appears in a sentence.

💡 The output is True or False, so you use it directly in if statements: if city in trip_list: or while "quit" not in commands:. Reads almost like plain English.
OperatorMeaningInput typesOutput typeExampleResult
inValue is presentany + sequencebool"Siddhesh" in friendsTrue/False
not inValue is absentany + sequencebool5 not in [1, 2, 3]True
python
friends = ["Siddhesh", "Sagar", "Rohan"]
print("Siddhesh" in friends)     # True
print("Alice" not in friends)     # True

# Works on strings too — checks for substring
print("chess" in "I enjoy chess and travelling")  # True

🔢 Bitwise Operators

Every number in your computer is stored as 0s and 1s (binary). Bitwise operators work directly on those 0s and 1s instead of the number itself. As a beginner you won't need these much, but they appear in competitive programming, hardware projects, and NumPy.

📝 These only work on whole numbers (int) and they give back a whole number. Don't use them as conditions in if or while. They're a completely different tool from the logical operators.
OperatorNameInput typesOutput typeExampleResult

>>Right shiftintint5 >> 12

5 in binary is 101, 3 is 011. Bitwise AND gives 001 = 1. Left shift by 1 is the same as multiplying by 2. Right shift divides by 2.

📐 Operator Precedence (BODMAS / PEMDAS in Python)

Remember BODMAS from school? Python works the same way. When you write something like 2 + 3 * 4, Python doesn't just go left to right. It follows a fixed order: some operators run before others. Here's the full order, from highest priority to lowest.

PriorityOperator(s)Category
1 (highest)( )Parentheses: always first, same as BODMAS brackets
2**Exponentiation (the "O" in BODMAS / "E" in PEMDAS)
3+x   -x   ~xUnary: positive, negative, bitwise NOT on a single value
4*   /   //   %Multiplication and Division
5+   -Addition and Subtraction
6<<   >>Bitwise Shift
7&Bitwise AND
8^Bitwise XOR
9|Bitwise OR
10==   !=   >   <   >=   <=   is   is not   in   not inComparison, Identity, Membership
11notLogical NOT
12andLogical AND
13 (lowest)orLogical OR

Simple rule: if you're ever unsure which part runs first, just wrap that part in parentheses. Parentheses always win. 2 + 3 * 4 gives 14, but (2 + 3) * 4 gives 20. When in doubt, be explicit.

🎤 These questions are commonly asked in Python interviews, especially in the first round. Click each question to expand the answer.
Beginner Level
Q1. What is the difference between / and // in Python? Beginner ›
One-line answer
/ always gives a float. // is floor division: it drops the decimal and rounds down toward negative infinity.
  • 10 / 2 gives 5.0, not 5. The result is always a float, even when it divides evenly.
  • 10 // 3 gives 3. The decimal is dropped and the result rounds down.

The "rounds down" part catches people off guard with negative numbers:

-7 // 2   # gives -4, not -3
# because -3.5 rounded DOWN is -4 (toward negative infinity)
Common trap
Expecting -7 // 2 to give -3 because "drop the decimal". Python floors toward negative infinity, not toward zero. So the result is always the next lower integer.
Q2. What does 10 % 3 return and why? Beginner ›
One-line answer
It returns 1. The % operator gives the remainder after integer division. 10 divided by 3 is 3 remainder 1.

The math: 10 = (3 x 3) + 1, so the remainder is 1.

Practical uses of modulo:

  • Even/odd check: n % 2 == 0 is True for even numbers
  • Wrap around a range: index % len(lst) keeps an index in bounds
  • Time conversion: total_seconds % 60 gives the leftover seconds
  • Every Nth item: if i % 5 == 0: triggers every 5 iterations
⚡ Say it in an interview
Modulo gives the remainder. The most common use cases are even/odd detection, cyclic indexing, and time/unit conversions.
Q3. What are augmented assignment operators? Give two examples. Beginner ›
One-line answer
Augmented assignment operators do an operation and assign the result back in one step. x += 5 is shorthand for x = x + 5.

Every arithmetic operator has an augmented version:

  • x += 5 adds 5 to x
  • x -= 3 subtracts 3
  • x *= 2 multiplies by 2
  • x //= 4 floor divides by 4
  • x **= 2 squares x
  • x %= 3 replaces x with x mod 3

They are especially common inside loops where you accumulate a running total or counter.

⚡ Note on mutability
For mutable types like lists, += modifies in place. For immutable types like int and str, it creates a new object and rebinds the variable. The behaviour looks the same from outside but is different internally.
Q4. What will not not True evaluate to? Beginner ›
One-line answer
True. Double negation cancels out. not True is False, then not False is True.

Step by step: not not True = not False = True.

This pattern is occasionally used to convert any truthy/falsy value to a strict boolean:

not not []      # False  (empty list is falsy)
not not [1, 2]  # True   (non-empty is truthy)

That said, bool(x) is the cleaner way to do the same thing and is much easier to read.

Common trap
Thinking not not x returns the original value. It returns a strict True or False, not the original object. Compare with or/and, which return the actual operand value.
Intermediate Level
Q5. What is the difference between == and is? Intermediate ›
One-line answer
== checks if two values are equal. is checks if two variables point to the exact same object in memory. They are not the same thing.
a = [1, 2, 3]
b = [1, 2, 3]

a == b   # True  — same contents
a is b   # False — different objects in memory

c = a
a is c   # True  — c points to the exact same object as a

is compares identity (memory address via id()). == compares value by calling the object's __eq__ method.

⚡ Rule
Use is only for None, True, and False (which are singletons). Use == for everything else. Writing x is "hello" may work sometimes but is not reliable.
Q6. What is short-circuit evaluation in Python? Intermediate ›
One-line answer
Python stops evaluating a logical expression the moment the result is certain. With and, a falsy left side skips the right. With or, a truthy left side skips the right.
  • False and (any expression): the right side is never evaluated because the result is already False.
  • True or (any expression): the right side is never evaluated because the result is already True.

This is used as a safety guard in real code:

if user is not None and user.name == "Rohan":
    # If user is None, the second check is never reached
    # so no AttributeError is raised
⚡ Say it in an interview
Short-circuit evaluation means Python is lazy: it stops the moment the outcome is known. This is used for safety checks (guard clauses) and default value patterns like name = input() or "Guest".
Q7. What is operator precedence? Give an example where it matters. Intermediate ›
One-line answer
Operator precedence defines which operator runs first when multiple operators appear in one expression. Python follows BODMAS/PEMDAS, plus a defined order for logical operators.

Priority order (high to low): **, then unary -/+/not, then * / // %, then + -, then comparison operators, then not, then and, then or.

2 + 3 * 4       # 14, not 20 (* runs before +)
not False or True   # True, not False
# because not binds tighter than or:
# = (not False) or True = True or True = True
⚡ Best practice
When in doubt, use parentheses. They make the intent explicit, prevent bugs from precedence surprises, and make the code easier to read. Never rely on obscure precedence rules in production code.
Advanced Level
Q8. How does the in operator perform differently across data structures? Advanced ›
One-line answer
in is O(n) for lists and tuples, O(1) for sets and dicts. The data structure you choose for membership checks has a huge performance impact.
  • list / tuple: scans from the beginning until it finds the item or reaches the end. O(n). Slow for large collections.
  • set / dict: uses a hash table. Computes the hash, goes directly to the right bucket. O(1) average. Fast regardless of size.
  • str: checks for a substring match, not a character. "ell" in "hello" is True. Time complexity is O(n*m) in the worst case.
items_list = list(range(1_000_000))
items_set  = set(range(1_000_000))

999999 in items_list  # slow — scans up to 1 million elements
999999 in items_set   # fast — hash lookup
⚡ Say it in an interview
If you are doing repeated membership checks and performance matters, convert your list to a set first. The lookup goes from O(n) to O(1).
Q9. Why does 0 and (1/0) not raise a ZeroDivisionError? Advanced ›
One-line answer
Because of short-circuit evaluation. 0 is falsy, so Python already knows the and expression is False and never evaluates the right side at all.

With and: if the left operand is falsy, the result is definitely falsy. Python returns the left operand immediately and never touches the right side.

0 and (1/0)   # returns 0, no error raised
# (1/0) is never evaluated

This is exactly why short-circuit evaluation is used as a safety guard. The dangerous or expensive operation on the right only runs when needed.

⚡ Say it in an interview
Short-circuit means Python is lazy. With and, once the left side is falsy, the right side is skipped entirely. That is why guard patterns like if lst and lst[0] == x are safe even when lst is empty.
Q10. What is the result of -7 % 3 in Python, and why does it differ from C/Java? Advanced ›
One-line answer
Python gives 2. C/Java give -1. The difference is which direction each language rounds: Python floors toward negative infinity, C/Java truncate toward zero.

Python's rule: the result of a % b always has the same sign as the divisor (b, the right-hand number).

C/Java rule: the result has the same sign as the dividend (a, the left-hand number).

-7 % 3 in Python:
  -7 = (3 * -3) + 2   → remainder is 2  (positive, sign of divisor 3)

-7 % 3 in C/Java:
  -7 = (3 * -2) + -1  → remainder is -1 (negative, sign of dividend -7)
⚡ Say it in an interview
Python's modulo result always has the same sign as the divisor. This comes from using floor division internally. C/Java use truncation toward zero instead, giving a different sign for negative operands.
⚠️ These are the most common operator mistakes beginners make. Click each one to see the fix.
1
Using = instead of == in a comparison
SyntaxError
›
Why it happens: = is the assignment operator. == is the comparison operator. In most programming languages, confusing these only gives wrong results. In Python, using = inside an if condition is a SyntaxError, so Python actually catches it for you.
❌ Wrong
python
if score = 100:
    print("Perfect!")
Python says
SyntaxError: invalid syntax
(Maybe you meant '==' instead of '='?)
✅ Correct
python
if score == 100:
    print("Perfect!")
🧠 Memory Trick
One equals sign is a command: "set this to that." Two equals signs is a question: "is this equal to that?" When you are asking a question inside an if, you always need the question form, which is ==.
⚡ Key Rule
= assigns a value. == checks if two values are equal. Inside if, while, or any condition, always use ==.
2
Using is instead of == to compare values
Logic Error
›
Why it happens: is checks if two variables point to the same object in memory, not if they have the same value. Python secretly caches small integers (-5 to 256), so a is b might return True for small numbers and False for large ones. Same value, different results: very confusing.
❌ Wrong for values
python
a = 1000
b = 1000
print(a is b)
Actual output
False
(same value, but different objects in memory!)
✅ Use == for value comparison
python
a = 1000
b = 1000
print(a == b)  # True (values match)

# is is only for None, True, False
if x is None: ...
🧠 Memory Trick
Two identical twins look the same (==) but are not the same person (is). When you want to ask "do these have the same value?", use ==. When you want to ask "are these literally the same object in memory?", use is.
⚡ Key Rule
Use == to compare values. Reserve is only for None, True, and False. Never use is to compare numbers, strings, or lists.
3
Using // when you needed a decimal result
Logic Error
›
Why it happens: // is floor division: it throws away the decimal and gives only the whole number. If you accidentally type // instead of /, your averages, percentages, or rates will be silently wrong with no error thrown.
❌ Loses the decimal
python
avg = 7 // 2
print(avg)
Actual output
3
(expected 3.5, got 3 — decimal silently dropped)
✅ Use / for decimal results
python
avg = 7 / 2
print(avg)  # 3.5

# // is useful only when you want whole number
pages = 100 // 3  # 33 full pages
🧠 Memory Trick
One slash / is the normal road: it takes you all the way to the answer including the decimal. Two slashes // are a speed bump that stops you at the whole number and drops everything after. More slashes, less precision.
⚡ Key Rule
/ always gives a float (even 6/2 gives 3.0). // always drops the decimal. Use / for math, use // only when you deliberately want an integer result.
4
Forgetting operator precedence
Logic Error
›
Why it happens: Python has a fixed order in which it evaluates operators, just like maths (BODMAS/PEMDAS). ** runs before *, which runs before +. If you forget this, the expression gets calculated in an order you did not intend.
❌ Wrong assumption
python
val = 2 + 3 ** 2
print(val)
Actual output
11
(** runs first: 3**2=9, then 2+9=11, not 25)
✅ Use parentheses to be explicit
python
val = (2 + 3) ** 2
print(val)  # 25
🧠 Memory Trick
When in doubt, add parentheses. They cost nothing, make the code easier to read, and guarantee Python does exactly what you intended. Even if the order is correct without them, parentheses make your intent obvious.
⚡ Key Rule
Python's precedence order (high to low): ** then * / // % then + -. Use (parentheses) whenever you mix operators to make the order explicit.
5
Using and/or instead of &/| with NumPy
ValueError
›
Why it happens: and/or are designed for single True/False values. NumPy arrays contain many values, and Python does not know which one to use for the boolean check. It raises a ValueError. The bitwise operators & and | work element-by-element instead.
❌ Wrong with arrays
python
import numpy as np
arr = np.array([1, 2, 3, 4])
arr[arr > 1 and arr < 4]
Python says
ValueError: The truth value of an array
with more than one element is ambiguous.
✅ Use & and | instead
python
arr[(arr > 1) & (arr < 4)]
# [2, 3] — works element by element
🧠 Memory Trick
and/or are for humans making one decision. &/| are for machines working on many values at once. NumPy is always working on many values, so always use the machine operators. Also: always wrap each condition in its own parentheses when using &/|.
⚡ Key Rule
With NumPy and pandas: use & (and), | (or), ~ (not). Always wrap each condition in parentheses: (arr > 1) & (arr < 4).
6
Modulo with negative numbers gives unexpected results
Logic Error
›
Why it happens: Python's % always returns a result with the same sign as the divisor (the number on the right). This is different from C, Java, and JavaScript where the result follows the sign of the dividend. If you came from another language, your intuition here will be wrong.
❌ Surprising results
python
print(-7 % 3)   # expected -1, got 2
print(7 % -3)   # expected 1, got -2
Why Python gives 2
Python floors towards negative infinity.
-7 = (-3 * 3) + 2, so remainder is 2.
✅ Force positive remainder
python
# If you always want a positive result:
print(abs(-7 % 3))   # 2
print(abs(-7) % 3)   # 1 (negate first)
🧠 Memory Trick
Remember: Python's modulo result borrows the sign from the divisor (right side). If the divisor is positive, the result is positive. If the divisor is negative, the result is negative. The left side does not decide.
⚡ Key Rule
Python's % result always has the same sign as the divisor. For a guaranteed positive remainder, apply abs(), or negate the dividend before taking modulo: abs(n) % d.
Key Takeaways
➕Arithmetic7 operators
  • / always returns a float, even for whole results (6/2 = 3.0)
  • // floor divides: drops the decimal, always gives int
  • % gives remainder, ** is exponent (not ^)
10 / 3 = 3.333 | 10 // 3 = 3 | 10 % 3 = 1
⚖️Comparison6 operators
  • Always return True or False
  • Never mix up = (assign) and == (compare)
  • Chaining works: 0 <= x <= 100 is valid Python
if 0 <= score <= 100: # chained comparison
🧠Logicaland / or / not
  • Combine conditions into one expression
  • Python short-circuits: stops early when result is clear
  • and needs both True; or needs only one True
if age >= 18 and has_id: # both must be True
📌Assignment shortcutsoperators
  • x += 5 is the same as x = x + 5
  • Works with all arithmetic operators: -=, *=, /=, **=, etc.
  • Very common in loops and running totals
score += 10 | total *= 2 | count -= 1
🔍Identityis / is not
  • Checks if two names point to the exact same object in memory
  • Use only for None, True, False
  • Never use is to compare numbers or strings
if result is None: ... # correct use of is
🎯Membershipin / not in
  • Checks if a value exists inside a sequence
  • Works on lists, tuples, strings, dicts, sets
  • O(1) for sets and dicts, O(n) for lists
if "a" in my_list: | if key in my_dict:
📐Precedenceorder of operations
  • Same idea as BODMAS/PEMDAS but extended
  • Order: ** then * / // % then + - then comparisons then not and or
  • When unsure, always add parentheses
2 + 3 ** 2 = 11 | (2+3) ** 2 = 25
🔢Bitwiseadvanced
  • Work on the binary (0s and 1s) representation of integers
  • & AND, | OR, ^ XOR, ~ NOT, << left shift, >> right shift
  • Required when using NumPy/pandas filters instead of and/or
(arr > 1) & (arr < 4) # NumPy filtering
